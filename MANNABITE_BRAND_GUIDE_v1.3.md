# 🍞 MannaBite — Brand Guide v1.3

> **작성일**: 2026-05-28 (v1.3)
> **서비스명**: MannaBite
> **태그라인**: "말씀을 한 입씩, 매일 맛보다"
> **English tagline**: "God's Word, one bite at a time"

---

## 1. 브랜드 스토리

### 이름의 의미

**Manna** (만나): 출애굽기 16장에서 하나님이 이스라엘 백성에게 하늘에서 내려주신 양식. 매일 아침 필요한 만큼만 거두어 먹었던 기적의 빵.

**Bite** (바이트): 한 입 크기. 작고 소화하기 쉬운 단위.

**MannaBite** = 하늘의 말씀을 매일 한 입씩 맛보는 경험. 아이들이 부담 없이, 재미있게, 매일 조금씩 성경을 암송하는 것을 표현.

### 브랜드 성격 (Brand Personality)

| 특성 | 설명 | 느낌 |
|------|------|------|
| **Friendly** | 아이가 처음 보고도 친근하게 느끼는 | 동글동글, 부드러운 곡선 |
| **Playful** | 학습이 아닌 놀이처럼 느끼게 하는 | 밝은 색상, 이모지, 애니메이션 |
| **Trustworthy** | 부모가 안심하고 맡길 수 있는 | 깔끔한 레이아웃, 연구 기반 |
| **Encouraging** | 실패해도 괜찮다고 말해주는 | 긍정적 피드백, 성장 보상 |
| **Gender-neutral** | 남녀 모두 좋아하는 | 오렌지+그린 따뜻한 중성 팔레트 |

> **참고**: 팔레트·UI는 성별 중립. 캐릭터는 성경 인물(MVP는 다윗=남성)이지만, 로스터에 에스더(여성) 등을 더해 **캐릭터 차원의 성별 균형**을 맞춘다. (섹션 7-4)

### 톤 & 보이스

- **아이에게**: "잘했어!", "다시 해볼까?", "거의 다 왔어!" — 격려하는 친구 같은 톤
- **부모에게**: "과학적으로 검증된 암기법", "아이의 학습 현황을 확인하세요" — 신뢰감 있는 톤
- **절대 하지 않는 말**: "틀렸어", "못했어", "다시 해" — 부정적 표현 배제

## 2. 로고

### 로고 컨셉

**Primary Mark**: 만나 빵(둥근 빵) + 한 입 베어 먹은 모양 + 반짝이는 빛

만나는 성경에서 "작고 둥글며 서리 같은" 것으로 묘사됨 (출 16:14). 둥근 빵에서 한 입 베어 먹은 형태가 "Bite"를 자연스럽게 표현하고, 빵 위의 반짝임(✨)이 하늘에서 내려온 신비로운 양식을 상징.

### 로고 변형

```
[Primary]    🍞✨ MannaBite         ← 아이콘 + 워드마크 (기본)
[Icon only]  🍞✨                    ← 앱 아이콘, 파비콘
[Wordmark]   MannaBite              ← 텍스트만 (좁은 공간)
[Korean]     🍞✨ 만나바이트          ← 한국어 병기 시
```

### 로고 사용 시 Canva/디자인 프롬프트

```
Logo concept: A cute round bread (manna) with a bite taken out of the right side. 
The bread has a warm golden-beige color with subtle texture.
Small sparkles (2-3) floating above the bread in honey orange (#F0932B) and sunflower yellow (#F9CA24).
Style: Minimal, rounded, friendly, suitable for children's app.
The bite mark creates a subtle crescent moon shape.
Background: transparent.
```

### 앱 아이콘 가이드

- 형태: 둥근 모서리 사각형 (iOS/Android 표준)
- 배경: 오렌지-골드 그라데이션 (#F0932B → #F9CA24)
- 중앙: 흰색 만나 빵 아이콘 (한 입 베어 먹은 형태)
- 반짝임: 밝은 골드(#F9CA24) 작은 별 2-3개

```

Body (본문):            14px / Regular (Pretendard)
Caption (보조):         11-12px / Regular (Pretendard)

### 한국어 구절 표기

- 폰트: 나눔명조 또는 Pretendard
- 줄간격: 1.7~1.8 (넉넉하게)
- 출처 표기: "시편 56:3 (개역한글)" — 항상 번역본 명시 (저작권 성명표시권)

### 영어 구절 표기

- 폰트: Nunito 또는 Pretendard — **Regular체 (이탤릭 사용하지 않음)**
- 줄간격: 1.6
- 색상: Text Secondary (#8B8A99) — 한국어보다 약간 연하게 구분
- 출처 표기: "Psalm 56:3 (WEB)" — 항상 번역본 명시
- 한/영 구절은 동등한 비중으로 취급 — 이탤릭으로 "부차적" 느낌 주지 않음

## 5. 컴포넌트 스타일

### Border Radius (모서리 둥글기)

```
Small (pill, badge):     20px (완전 둥글림)
Medium (button, input):  16px
Large (card):            16-20px
XL (screen container):   20-24px
```

→ "동글동글" 브랜드 느낌의 핵심. 날카로운 모서리는 **절대 사용하지 않음**.

### 버튼 스타일

```
[Primary]   배경: #F0932B, 글자: #FFFFFF, radius: 16px, padding: 14px
[Ghost]     배경: transparent, 테두리: rgba(240,147,43,0.2), 글자: #E58E26
[Disabled]  opacity: 0.35, pointer-events: none
```

- 버튼 최소 높이: 48px (아이 손가락 터치 고려)
- 터치 시: scale(0.97) 효과
- 틀렸을 때: shake 애니메이션 (0.3초)

### 카드 스타일

```
배경: #FFFFFF
테두리: 0.5px solid rgba(240,147,43,0.1)
radius: 16px
padding: 14px
그림자: 없음 (플랫 디자인)
```

- Accent 카드: 배경에 tint color 사용 (오렌지틴트, 그린틴트, 골드틴트)
- 클릭 가능한 카드: cursor: pointer + 미세한 hover 효과

### 프로그레스 바

```

배경: #FFF3E0 (오렌지 틴트)

### Pill / Badge

```

radius: 20px (완전 둥글림)
font-size: 11px, font-weight: 500
```

- Fire pill: 배경 #FFF8E1, 글자 #F0932B

## 7. 캐릭터 시스템 (2레이어)

> **v1.3 변경**: 캐릭터를 **2레이어**로 분리 — (A) 아바타("나", 영구) + (B) 성장 캐릭터(다윗, phase 콘텐츠).
> **v1.2 변경**: 기존 "말씀 전사 + 에베소서 갑옷"을 성경 인물 성장 시스템으로 교체.

### 7-0. 두 레이어 개요

| 레이어 | 무엇 | 지속성 | 비주얼 | 비고 |
|--------|------|--------|--------|------|
| **A. 아바타** | "나" (아이 자신) | 영구 | **얼굴만** (원형 아이콘) | 누적 암송 **레벨**이 여기 붙음(레벨 UI는 Phase 2) |
| **B. 성장 캐릭터** | phase별 성경 인물 | 10일마다 교체 | 풀 일러스트(3단계) + 아이템 | MVP = 다윗 |

→ 아트 예산 분배: **아바타는 가볍게(얼굴), 성장 캐릭터에 공들이기(다윗).**

### 7-A. 아바타 ("나") — 가벼운 버전 MVP

- 아이가 **첫 진입 시 1회 선택**하는 자기 분신. 계정 없이 `localStorage` 저장.
- **얼굴만** — 풀바디/포즈 없음. 홈 헤더에 작은 원형 아이콘.
- **6개 옵션** (성별 강요 없이 아이가 선택 → gender-neutral):

| id | 이름 | 이모지 | 타입 |
|----|------|--------|------|
| boy | 남자아이 | 👦 | 사람 |
| girl | 여자아이 | 👧 | 사람 |
| lamb | 양 | 🐑 | 동물(성경 테마) |
| lion | 사자 | 🦁 | 동물(유다 지파) |
| dove | 비둘기 | 🕊️ | 동물(성령/평화) |
| rabbit | 토끼 | 🐰 | 동물(친근함) |

- 비주얼 방향: 둥글둥글, 밝은 표정, 따뜻한 팔레트. 6개가 **한 세트로 통일감** 있게(같은 선 굵기·스타일).
- 이미지 준비 전 개발 초기엔 이모지로 대체 렌더 가능.
- **MVP에서는 레벨·꾸미기 없음.** 아바타 레벨링/아이템은 Phase 2.

### 7-B. 성장 캐릭터 — 다윗 (10일 phase)

아이는 **10일 챌린지** 동안 한 명의 성경 인물을 키운다. 매일 학습 완료 시 아이템을 하나씩 모으고, 인물이 단계적으로 성장한다. **MVP 캐릭터는 다윗** — 목동 소년에서 시작해 왕이 되는 여정.

**왜 갑옷에서 바뀌었나**:
- 단일 아바타+갑옷보다 **실제 성경 인물의 서사**가 정서적 몰입이 큼
- **콘텐츠 확장성** — 에스더·모세 등 캐릭터를 계속 추가 가능
- **구절과 독립적인 게이미피케이션 레이어** — 어떤 구절을 외우든 다윗은 10일에 걸쳐 성장 (구절 순서·내용과 무관)

### 7-2. 성장 3단계 (다윗)

| 단계 | 이름 | 시기 | 비주얼 방향 |
|------|------|------|-------------|
| 1 | 목동 소년 다윗 | Day 1–3 | 어린 소년, 양치기 옷, 작은 체구, 맑고 밝은 눈 |
| 2 | 청년 다윗 | Day 4–7 | 조금 자란 청소년, 자신감 있는 표정, 물맷돌 |
| 3 | 왕 다윗 | Day 8–10 | 따뜻한 왕, 소박한 왕관, 위엄 + 친근함 공존 |

- 단계 전환: 부드러운 페이드/모핑. 과한 연출 금지.
- 세 단계 모두 **동일 인물로 인지**되게 핵심 앵커 유지 — 얼굴형, 머리색, 눈 모양, 피부톤, 의상 색 계열.
- 표정: 항상 밝고 긍정적.

### 7-3. 다윗 아이템 10개 (하루 1개 해금) — ⚠️ 제안 초안

> 아래는 **제안 초안**이며 아트 단계에서 확정. 슬롯 구조(Day N 완료 → slot N 해금)만 고정.
> 4–8세 대상이라 폭력적 요소(칼, 전쟁 등)는 **의도적으로 배제**하고 다윗 서사의 따뜻한 면을 택함.

| Day | 아이템 | 성경 근거 | 단계 |
|-----|--------|-----------|------|
| 1 | 🐑 양 | 다윗은 목동이었음 (삼상 16:11) | 목동 |
| 2 | 🦯 목동 지팡이 | 양 떼를 지킴 | 목동 |
| 3 | 🎵 수금 | 사울 앞에서 연주 (삼상 16:23) | 목동 |
| 4 | 🪨 물맷돌 | 골리앗 앞에서 (삼상 17:40) | 청년 |
| 5 | 🫙 기름병 | 사무엘이 기름 부음 (삼상 16:13) | 청년 |
| 6 | 🤝 친구 요나단 | 깊은 우정 (삼상 18:1) | 청년 |
| 7 | 🛡 방패 | 하나님의 보호 | 청년 |
| 8 | 📜 시편 두루마리 | 다윗이 시편을 지음 | 왕 |
| 9 | 📦 법궤 | 예루살렘으로 모셔옴 (삼하 6) | 왕 |
| 10 | 👑 왕관 | 이스라엘의 왕 (피날레) | 왕 |

**아이템 표시**:
- 미획득: 흐릿하게 (opacity 0.4) + ❓ 아이콘
- 획득: 밝은 컬러 + 골드 테두리
- 해금 연출: glow + bounce (0.5초)

**아이콘 스타일**:
- 둥글고 플랫, 따뜻한 팔레트, 라인+면 혼합
- 작게·빠르게 그릴 수 있는 단순 형태 (10개 × 캐릭터 수만큼 필요하므로 제작 효율 중요)

### 7-4. 캐릭터 로스터 (이후)

MVP는 **다윗 1명**. 컬렉션/완료 화면에 *"다음 캐릭터 곧 추가!"* 안내만 노출. 이후 에스더(여성)·모세 등을 추가해 다양성과 성별 균형 확보. 각 캐릭터 = 자체 성장 3단계 + 아이템 10개.

### 7-5. 일러스트 일관성 (제작 메모)

- 한 캐릭터가 3단계로 보이도록 **레퍼런스/시드 고정** (얼굴·비율·색 동일).
- 배경은 단순 또는 투명 — **캐릭터가 주인공**.
- 캐릭터는 저작권/일관성을 위해 직접 드로잉 우선, 필요 시 AI 보조 (배경 위주).

## 8. 모션 & 인터랙션

### 기본 원칙

- 모든 전환: 0.15~0.3초 (빠르고 부드럽게)
- 아이용이므로 **피드백이 즉각적**이어야 함
- 정답: 초록 배경 + 부드러운 확대
- 오답: shake 애니메이션 (0.3초) + 코랄 배경 → 바로 복구
- 완료: 🎉 confetti 또는 큰 이모지 + scale up 애니메이션

### 주요 인터랙션

```
버튼 터치:     scale(0.97) → 복구 (0.15초)
정답 선택:     배경 #A8E6CF fade in (0.2초)
오답 선택:     translateX ±4px shake (0.3초) + 배경 #FFF0EE
페이지 전환:   fade + slide left/right (0.25초)
프로그레스:    width transition (0.4초, ease-out)
완료 축하:     scale 0→1 bounce (0.5초)
아이템 해금:   glow effect + bounce (0.5초)
캐릭터 성장:   단계 전환 시 부드러운 페이드/모핑 (0.4초)
```
