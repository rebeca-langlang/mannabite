# 📖 MannaBite — 서비스 기획서 v1.2 (MVP 개발용)

> **작성일**: 2026-05-28
> **상태**: MVP 개발 착수
> **서비스명**: MannaBite — "말씀을 한 입씩, 매일 맛보다"
> **이 문서의 목적**: Claude Code 개발 세션에 공유하는 **MVP 구현 스펙**. 비즈니스 기획(시장분석·KPI 등)은 압축하고, **무엇을 만들고 무엇을 안 만드는지**를 명확히 한다.

## 0. 개발 세션 사용 안내 (먼저 읽기)

- 이 문서는 **MVP 한 사이클**(다윗 10일 챌린지 + 구절 10개)을 만드는 데 필요한 것만 담는다.
- **MVP는 프론트엔드 전용**이다. 백엔드/DB/인증/결제 **없음**. 진행 상태는 전부 `localStorage`.
- 오디오(TTS·챈트)와 이미지는 **사전 생성된 정적 파일**(`/public`)을 불러온다. 런타임 생성 없음.
- 섹션 **2(MVP 범위)**와 **14(제외 항목)**를 먼저 보고, 범위 밖 기능은 **절대 구현하지 말 것**.
- 데이터 구조는 섹션 **5(JSON 스키마)**, 디자인은 섹션 **12(디자인 토큰)** 기준.

## 1. 한 줄 요약 & 핵심 가치

게임형으로 성경 말씀을 **한국어로 이해하고 영어로 암송**하는 4–8세 어린이용 앱. 노래·그림·게임으로 "또 하고 싶다"를 만든다.

> "검증된 암기법(그림·노래·청크·인출·기도)을 게임으로 녹인, 한/영 이중언어 어린이 성경 암송 앱"

**타겟**: 4–8세 (sweet spot 4–7세) / 결제·관리자는 부모(주로 엄마) / 가정 신앙교육·주일학교 연계·영어 병행.

## 2. ⭐ MVP 범위 (가장 중요)

### 2-1. MVP에 포함 (이번에 만든다)

| 영역 | 내용 |
|------|------|
| **콘텐츠** | 큐레이션 구절 **10개** (한/영), 테마 순서 고정 (시 56:3 → … → 잠 3:5) |
| **학습 플로우** | 구절당 **6단계**: 그림 → 노래 → 청크 → 게임 → 기도 → 완료 |
| **게임** | **2종만**: ① 빈칸 채우기(단어 터치) ② 초성/첫글자 힌트 |
| **오디오** | 구절별 한/영 TTS(ElevenLabs, 사전생성) + 챈트 1곡(Suno, 사전생성) |
| **그림 연상** | 구절별 이모지 시퀀스 + 쉬운 설명(한국어) |
| **챌린지** | **다윗 10일 챌린지**(단일 캐릭터): 하루 1구절, 다윗이 10일간 성장 |
| **캐릭터 시스템** | 다윗 **성장 3단계** 일러스트 + **아이템 10개**(하루 1개 해금) |
| **챈트 라이브러리** | 모은 챈트를 언제든 재생(차/취침 시간용) |
| **복습(경량)** | "오늘 복습할 구절" 1개를 가볍게 제시(빈칸 1회), 건너뛰기 가능·페널티 없음 |
| **동기부여** | 출석 스트릭(🔥) + 별/XP 단순 카운터 + 완료 축하 애니메이션 |
| **컬렉션 화면** | 다윗 아이템 도감(획득/미획득) |
| **저장** | `localStorage` (진행도·스트릭·획득 아이템·챈트) |
| **플랫폼** | 모바일 우선 반응형 웹 (태블릿도 모바일 레이아웃) |

### 2-2. MVP의 핵심 루프

```
홈에서 "오늘의 말씀" 탭
  → (선택) 어제 구절 가벼운 복습 1회
  → 6단계 학습 (그림→노래→청크→게임→기도)
  → 완료: 별 획득 + 다윗 아이템 1개 해금 + 챈트 1곡 라이브러리에 저장
  → 다윗 성장 단계 갱신
  → "내일 또 만나자!"
10일 반복 → 다윗 완전 성장(왕) + 구절 10개 + 챈트 10곡
```

## 3. 사용자 인터뷰 인사이트 (n=6, 개발 우선순위 근거)

> 개발 전 엄마 5명 + 내부 1명 설문 완료. **컨셉·기능 방향은 강하게 검증됨. 단, 지불 의향은 미측정.** (응답자는 우호적 지인 + 이미 암송 시키는 부모라 desirability 편향 있음)

| 항목 | 결과 | 개발 함의 |
|------|------|----------|
| 나이 | 3–6세 5명, 7–8세 1명 | 타겟 적중. 글 못 읽는 아이 전제로 UI(이모지+음성) 설계 |
| 가장 큰 어려움 | 부모 시간부족(4), 체계 없음(3), 지루함(3) | 문제 검증됨 → "체계 + 재미"가 코어 |
| 가장 매력적 기능 | **그림 연상(4) > 노래(3)** > 빈칸/테마/한영순서(각2) | **그림·노래에 최우선 투자.** 게임류는 과투자 금지(빈칸만 2표, 초성은 선택지에도 없었음) |
| 영어 관심 | 매우관심 5 + 어느정도 1 | 이중언어 강하게 먹힘 |
| 이중언어 방식 | "한국어 먼저 → 영어" 선호 | **한→영 순서를 기본값으로** (동시노출 X) |
| 이름 | MannaBite 5 / MannaSeed 1 | 이름 확정: **MannaBite** |
| 사용 의향 | 꼭 사용 5 / 한번 1 | 무료 기준 desirability. 가격 신호 아님 |

**자유 의견에서 나온 것** (전부 MVP 제외, 추적만):
- 아이 음성 녹음(1) → Phase 2
- 암송 영상 커뮤니티 공유(1) → Phase 3
- 여러 성경 버전(표준새번역 등, 1) → ⚠️ 저작권 이슈, Phase 3(라이선스 후)
- 중국어 버전(1) → 장기 해외 확장 시그널
- **"내가 원하는 구절 직접 선택"(1)** → 이 응답자는 7–8세 + 303비전학교 외부 커리큘럼 사용자. 핵심 타겟과 다른 상위 세그먼트. **MVP는 큐레이션 고수, 자유 입력 X.** Phase 3 "교회/프로그램 구절 연동" 페르소나로 보관.

**미측정 → 별도 테스트 필요**: 지불 의향(₩X/월). 소프트 런칭 후 가격 검증 To-do.

```json
  "version_en": "WEB",
  "text_ko": "내가 두려워하는 날에는 주를 의지하리이다",
  "text_en": "When I am afraid, I will put my trust in you.",
  "chunks_ko": ["내가 두려워하는 날에는", "주를 의지하리이다"],
  "chunks_en": ["When I am afraid,", "I will put my trust in you."],
  "emoji_sequence": ["😨", "🙏", "😊"],
  "emoji_labels_ko": ["두려움", "기도/의지", "평안"],
  "simple_explain_ko": "무서울 때 하나님을 믿고 의지해요.",
  "audio_ko": "/audio/ps56-3_ko.mp3",
  "audio_en": "/audio/ps56-3_en.mp3",

  "game_blanks_en": ["afraid", "trust"]

```

- `day`: 1–10. 학습 순서 = `day` 오름차순.
- `game_blanks_*`: 빈칸/초성 게임에서 가릴 핵심 단어(구절당 1–2개).
- `emoji_labels_ko`: 글 못 읽는 아이 + 접근성용 텍스트 라벨(이모지마다 1개).
- **챈트 가사 = `text_ko`/`text_en` 그대로**(verbatim). 별도 가사 필드 없음.

### 5-2. 캐릭터 (`character_david.json`)

```json
{
  "id": "david",
  "name_ko": "다윗",
  "stages": [
    { "stage": 1, "name_ko": "목동 소년 다윗", "day_from": 1,  "image": "/img/david/stage1.png" },
    { "stage": 2, "name_ko": "청년 다윗",     "day_from": 4,  "image": "/img/david/stage2.png" },
    { "stage": 3, "name_ko": "왕 다윗",       "day_from": 8,  "image": "/img/david/stage3.png" }
  ],
  "items": [
    { "slot": 1,  "name_ko": "(초안) 물맷돌", "icon": "/img/david/item01.png" },
    { "slot": 2,  "name_ko": "(초안) 수금",   "icon": "/img/david/item02.png" }
    /* slot 3~10 동일 구조. 이름·아이콘은 아트 단계에서 확정 */
  ]
}
```

- **성장 단계 규칙(기본값, config로 조정 가능)**: `day_from` 이상이면 해당 단계 이미지 표시. → Day 1–3 목동, 4–7 청년, 8–10 왕.
- **아이템 해금 규칙**: Day N 완료 시 `slot === N` 아이템 해금. (Day 10 = 마지막 아이템, "왕관"으로 마무리 권장)
- ⚠️ **아이템 10개 이름/아이콘은 아직 미확정(초안).** 슬롯 구조만 고정. 아트 작업 시 확정 → 이 JSON만 갱신하면 됨.

## 6. 핵심 학습 플로우 (6단계)

> 순서 근거: Mayer의 멀티미디어 학습 인지이론(2001). **그림(의미 스키마)을 먼저** 형성 → 그 위에 노래(청각) 부호화. 4–8세는 사전지식이 적어 Pre-training 효과가 큼. 1회 세션 약 5–7분.

```
① 그림으로 이해하기 (Dual Coding · Pre-training)
   - 이모지 시퀀스 + 쉬운 설명(한국어)
   - 역할: 의미 스키마 형성

② 노래로 만나기 (Musical Encoding)
   - 챈트 재생 + 가사 하이라이팅, 한국어 먼저 → 영어
   - 가사 = 구절 그대로

③ 청크로 쌓기 (Chunking)
   - 구절을 2~4개 청크로 분할, 하나씩 듣기→따라 말하기→연결
   - 한국어 → 영어 순서

④ 게임으로 외우기 (Active Recall)   ← 게임 2종만 (섹션 7)
   - 빈칸 채우기 → (통과 시) 초성/첫글자 힌트
   - 틀려도 페널티 없음, "다시 해보자" 톤

⑤ 기도로 새기기 (Depth of Processing · Self-Reference)
   - 구절 키워드를 담은 짧은 기도문(한/영), 따라 읽기 + TTS
   - 저비용(텍스트 + 기존 TTS 재사용). 새 시스템 추가 아님

⑥ 오늘의 완료!
   - 별 +N, 다윗 아이템 1개 해금, 챈트 라이브러리 저장
   - 스트릭 갱신, "내일 또 만나자!"
```

> **단계 인디케이터**: 학습은 5스텝(①~⑤), 완료(⑥)는 결과 화면. UI 상단에 `Step X/5` 표기.

## 7. 게임 상세 (2종만)

### Game 1 — 빈칸 채우기
- 구절을 청크로 보여주고 `game_blanks`의 핵심 단어를 빈칸 처리.
- 하단 단어 칩(정답 + 오답 2–3개)을 **터치**해서 채움.
- 피드백: 정답 칩 → Sage Green 배경 fade-in / 오답 → Soft Coral + 0.3초 shake 후 복구.
- 한국어 라운드 → 영어 라운드.

### Game 2 — 초성/첫글자 힌트
- 한국어: 각 단어의 **초성만**(ㄱㄴㄷ) 보여주고 떠올려 말하기/선택.
- 영어: 각 단어의 **첫 글자**만 보여줌.
- 4–8세 난이도 고려: 자기 확인(self-confirm) 방식 허용("기억났어요!" 버튼) — 음성 인식 X(Phase 2).

> **순서 맞추기·전체 암송(타이핑/음성) 게임은 MVP 제외.** 인터뷰에서 게임류 선호 낮았고, 코어는 그림·노래.

## 8. 다윗 10일 챌린지 시스템

- **구절과 캐릭터는 분리**된다. 다윗은 **시각/게이미피케이션 레이어**일 뿐, 어떤 구절을 외우는지와 무관.
- 구절 순서는 테마/난이도 기준(짧음→긺) 그대로 유지(섹션 13).
- **하루 1구절 = 1일치.** Day N 완료 시:
  1. 다윗 아이템 slot N 해금
  2. 챈트 라이브러리에 해당 챈트 추가
  3. 다윗 성장 단계 갱신(day_from 규칙)
- 10일 완료 = 다윗 "왕"까지 성장 + 구절 10개 + 챈트 10곡 + 아이템 10개.
- **MVP는 다윗 단일 캐릭터.** 컬렉션/완료 화면에 *"다음 캐릭터 곧 추가!"* 안내만. (에스더·모세 등은 이후 콘텐츠로 추가)

## 9. 챈트 라이브러리

- 학습으로 **해금된 챈트**를 모아두는 화면. 언제든 재생(차 안·취침 전 반복 청취용).
- 항목: 구절 레퍼런스 + 테마 + 재생 버튼 + (선택) 즐겨찾기 하트.
- 한/영 토글 재생.
- **복습 시스템과 별개**다. 챈트함은 "영구 보관함", 복습은 "오늘 할 것".

## 10. 복습 — MVP 경량 버전

> ⚠️ **MVP는 본격 간격반복(SM-2 등) 알고리즘을 만들지 않는다.** 가벼운 고정 오프셋만.

- 새 구절 학습 후 `reviewQueue`에 고정 간격으로 등록: **+1일, +3일, +7일** (3회면 충분).
- 매일 홈/학습 진입 시 "오늘 복습할 구절" 최대 1개를 **가볍게**(빈칸 1회) 제시.
- **건너뛰기 가능, 페널티/스트릭 차감 없음.** 부담 주지 않기.
- 전체 SM-2식 적응형 스케줄 + 부모 대시보드 통계는 **Phase 2**.

## 11. 화면 명세

> 디자인 토큰은 섹션 12. 모든 터치 타겟 ≥ 48px, 이모지 선택지 ≥ 60px.

### 11-1. 홈
- 상단: 🔥 스트릭(출석 N일), 다윗 현재 모습(성장 단계 이미지) + 단계명.
- 메인 CTA: **"▶ 오늘의 말씀"** (Day N / 테마 표시).
- (있으면) "📖 오늘 복습 1개" 가벼운 진입.
- 진행 바: `현재일 / 10`.
- 하단 탭(3개): **오늘** · **챈트함** · **다윗 도감**. (설정은 도감 화면 우상단 톱니로) → 탭 4개 이하 유지.

### 11-2. 학습 (Step 1/5 ~ 5/5)
- 상단: `Step X/5` + 단계명 + 뒤로가기.
- 본문: 단계별 콘텐츠(섹션 6).
- 한국어 🇰🇷 | English 🇺🇸 토글(노래·청크·기도 단계).
- 하단 고정 CTA: `다음 →` / 필요 시 `다시 듣기`.

### 11-3. 완료
- 🎉 "잘했어요!" + 오늘 구절 ✅.
- `+N ⭐` / 🔥 스트릭.
- **다윗 아이템 해금 연출**(glow + bounce) + 챈트 저장 안내.
- 버튼: `홈으로` / `챈트 듣기`.

### 11-4. 다윗 도감(컬렉션)
- 다윗 현재 모습(대형) + 성장 단계 라벨.
- 아이템 10칸 그리드: 획득=컬러+골드테두리 / 미획득=흐릿(opacity 0.4)+❓.
- 하단: *"다음 캐릭터 곧 추가!"*.

### 11-5. 챈트함
- 해금된 챈트 리스트(섹션 9).
